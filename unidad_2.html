Unidad 2: Procesos del Software



Procesos del Software

Es un conjunto estructurado de actividades necesarias para desarrollar un sistema de software.
Muchos de los procesos de software son diferentes, pero todos implican:
1.- Especificación
2.- Diseño e implementación
3.- Validación
4.- Evolución


Otras caracerísticas del proceso de SW


1.- Comprensión
2.- Visibilidad
3.- Soporte
4.- Aceptación
5.- Confianza
6.- Robustez
7.- Capacidad de almacenamiento
8.- Rapidez
9.- Adaptación

Estándares relacionados con el proceso de SW

Estándar ISO/IEC/IEEE 12207:2017

Se aplica a la adquisición de sistemas de software , productos y servicios, al suministro, desarrollo, operación, mantenimiento y
eliminación de productos de software o componentes de software de cualquier sistema.
Se incluyen aquellos aspectos de la definición del sistema necesarios para proporcionar el contexto de los productos y servicios de software.
También proporciona procesos que pueden emplearse para definir, controlar y mejorar los procesos del ciclo de vida del software dentro
de una organización o de un proyecto.

Ciclo de vida del Software


Cuando un proceso implica la construcción de algún producto, suele referirse al proceso como un ciclo de vida.
Los proyectos software se desarrollan en una serie de fases y estas fases son:

Temporales: Forman una secuencia en el tiempo

Lógicas: Cuando representan pasos o etapas que no constituyen una secuencia temporal

Se puede definir ciclo de vida del software como Las distintas fases por las que pasa el software desde que nace una necesidad
de mecanizar un proceso hasta que deja de utilizarse el software que sirvió para ese objetivo, pasando por las fases de desarrollo
y explotación [Frakes et al., 1991]

Ambito general del ciclo de vida del Software

Desde un punto de vista general puede considerarse que el ciclo de vida de un software tiene tres etapas claramente diferenciadas:
1.-Planificación
2.- Implementación
3.- Puesta en producción

Objetivos de cada etapa

1.- Especificaciones
2.- Análisis
3.- Diseño
4.- Implementación
5.- Debugging
6.- Validación
7.- Evolución

Modelo de Proceso de Software

Hay varios modelos de procesos definidos en la bibliografía de Ingeniería del Software. Cada modelo de proceso representa
un proceso desde una perspectiva particular, por lo que sólo ofrece una información parcial obre dicho proceso.

Modelo general del Proceso en Ingeniería

1.- Especificaciones
2.- Diseño
3.- Fabricación
4.- Prueba
5.- Instalación
6.- Mantenimiento
